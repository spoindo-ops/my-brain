# 🎥 최종 Visual Sign-off 및 QA 통합 문서

본 문서는 Writer가 제시한 스크립트와 Shot List에 대한 시각적 가이드라인(Visual System)을 개발팀의 QA 시스템과 완벽하게 연동하기 위해 최종 확정된 모든 규칙, 포맷, 및 승인 프로세스를 정의합니다. **모든 영상 에셋은 이 기준을 100% 준수해야 하며, '신뢰 자본' 구축의 핵심 요소로 기능합니다.**

## 1. 공식 시각 시스템 (Visual System) 최종 확정

모든 콘텐츠는 다음의 색상, 타이포그래피, 레이아웃 규칙을 **절대적으로** 따라야 합니다.

### 1.1. 컬러 팔레트 정의
| 용도 | 색상 이름 | HEX 코드 | 역할 및 강조 방식 |
| :--- | :--- | :--- | :--- |
| **주요 배경 (Primary BG)** | Deep Navy | `#000080` | 영상의 주된 배경, 신뢰감 극대화. |
| **강조 색상 (Accent Color)** | Gold | `#FFD700` | 핵심 메시지, 인증 마크, 중요한 텍스트 강조에 사용. |
| **보조 텍스트 (Secondary Text)** | White | `#FFFFFF` | Deep Navy 배경 위에서 가독성 확보. |

### 1.2. 타이포그래피 규칙 (Typography Rules)
*   **제목/핵심 메시지:** Gold 색상을 사용하여 시각적 충격을 극대화합니다. (예: `H1`, 핵심 후크 문구)
*   **본문 설명:** White 색상을 사용하여 명확한 정보를 전달합니다. (예: Shot List 설명, 보조 텍스트)
*   **폰트 선정:** 가독성과 프리미엄 느낌을 동시에 충족하는 산세리프 계열 폰트를 사용합니다. (예: Montserrat, Futura 또는 이와 동등한 고품질 폰트)

### 1.3. 레이아웃 및 구성 원칙
*   **골드 강조 원칙:** 모든 시각 요소에서 Gold는 **'인증', '품질 증명', '신뢰 자본'**을 나타내는 데만 사용되어야 합니다. 단순한 장식이 아닌, 품질의 증명으로 기능해야 합니다.
*   **Deep Navy 배경 활용:** Deep Navy는 콘텐츠에 깊이와 프리미엄함을 부여하며, Gold 강조 요소가 시선을 즉시 집중시키는 프레임 역할을 수행합니다.

## 2. Visual Sign-off QA 프로세스 및 포맷 (개발팀 연동)

이 섹션은 Writer/Designer의 승인 결과가 Developer의 QA 시스템에 자동으로 기록되도록 하는 표준화된 포맷을 정의합니다.

### 2.1. 최종 QA 체크리스트 (Master Checklist)
모든 영상 에셋(썸네일, Intro, 주요 컷) 제작 시, 다음 항목들에 대한 **'Yes/No'** 및 **'비고(Comment)'**를 기록해야 합니다.

| No. | 검토 항목 (Visual System 준수 여부) | 기준 (Deep Navy/Gold 적용) | 승인 상태 (Y/N) | 비고 (QA 기록) |
| :--- | :--- | :--- | :--- | :--- |
| **1** | **컬러 시스템 일관성** | Deep Navy 배경 및 Gold 강조가 전체 프레임에 적용되었는가? | | |
| **2** | **골드 사용 범위** | Gold는 오직 '인증' 또는 '핵심 품질 증명' 요소에만 사용되었는가? (과도한 사용 금지) | | |
| **3** | **타이포그래피 적용** | 확정된 폰트 및 색상 규칙(Gold/White)이 정확히 적용되었는가? | | |
| **4** | **신뢰 자본 반영** | '품질 인증' 또는 관련 시각적 요소가 명확하게 제시되었는가? (CEO KPI 연계 여부) | | |
| **5** | **레이아웃 적합성** | Shot List 및 영상 흐름에 따라 레이아웃이 논리적으로 구성되었는가? | | |

### 2.2. Visual Sign-off 기록 포맷 (Approval Log Format)
개발팀은 아래의 포맷을 사용하여 모든 자산에 대한 승인 기록을 데이터베이스에 저장해야 합니다.

**[Asset Approval Record]**

*   **Asset ID:** (DB에서 자동 생성 또는 수동 입력)
*   **Content Type:** (예: YouTube Intro / Reel Thumbnail / Product Shot)
*   **Creator:** (Writer / Designer)
*   **Version:** (V1.0, V1.1 등 버전 관리)
*   **Visual System Check:** (위 2.1 체크리스트 결과 요약)
    *   컬러 일관성: [Y/N] - 비고: [ ]
    *   골드 사용 범위: [Y/N] - 비고: [ ]
    *   타이포그래피 적용: [Y/N] - 비고: [ ]
    *   신뢰 자본 반영: [Y/N] - 비고: [ ]
*   **Sign-off Status:** (Approved / Rejected / Revision Needed)
*   **Final Approval Date:** YYYY-MM-DD
*   **Reviewer ID:** (승인한 담당자 ID)

## 3. 썸네일 컨셉 표준화 (Template Standardization)

앞으로 모든 썸네일은 다음 3가지 확정된 컨셉 중 하나를 기반으로 제작되어야 하며, 이는 **'Global Trust Series'**의 일관성을 유지합니다.

### 3.1. 썸네일 컨셉 3안 (최종 선택 및 적용)

| 컨셉 | 핵심 목표 | 시각적 특징 | 사용 예시 |
| :--- | :--- | :--- | :--- |
| **컨셉 A: The Gold Seal** | 최고 품질 인증 강조 (Trust Capital 극대화) | Deep Navy 배경 전체에 크고 명확한 골드 인증 마크 배치. 텍스트는 중앙 상단에 위치. | 시스템/제조 과정 공개 영상 |
| **컨셉 B: Focus on Result** | 제품 효능 및 결과 중심 제시 | 좌측 또는 우측에 고화질 제품 이미지(Before & After)를 크게 배치하고, Gold로 핵심 결과 수치를 강조. | 마스크팩 사용 후기 영상 |
| **컨셉 C: The Blueprint** | 제조 과정의 투명성 강조 (Supply Chain Trust) | Deep Navy 배경 위에 Shot List 또는 제조 흐름도를 미니멀하게 표현하고, Gold 선으로 흐름을 연결. | 마스크팩 제조 과정 공개 영상 |

### 3.2. 최종 적용 가이드라인
*   **선택:** **컨셉 A (The Gold Seal)**를 기본 템플릿으로 채택합니다. 이는 '신뢰 자본'이라는 브랜드 목표에 가장 직접적으로 부합하며, 최고 품질의 이미지를 즉각 전달합니다.
*   **프레임:** 모든 썸네일은 16:9 비율을 유지해야 하며, Gold 마크는 화면 중앙에서 약간 위쪽에 위치하여 시선이 자연스럽게 아래로 흐르도록 유도합니다.
