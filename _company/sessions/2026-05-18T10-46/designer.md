### 1. 'Deep Navy/Gold' 시스템 기반 자동화 목표 정의

**목표:** 영상 에셋(썸네일, 도입부 화면) 제작 시, 수동 검토 없이도 'Deep Navy/Gold' 시스템 준수 여부를 자동으로 검증하고 적용하는 **Visual Validator Module**을 구축한다.

**핵심 원칙:**
1.  **색상 일관성 (Color Consistency):** 모든 썸네일과 도입부 배경은 정의된 Deep Navy/Gold 색상 팔레트를 100% 준수해야 한다.
2.  **타이포그래피 계층 (Typography Hierarchy):** 핵심 메시지, 보조 정보, 인증 마크의 크기, 굵기, 간격(Spacing)이 사전에 정의된 규칙을 따라야 한다.
3.  **인증 요소 배치 (Trust Element Placement):** '품질 증명'을 나타내는 금색 강조 및 인증 마크의 위치와 크기는 고정되어야 한다.

### 2. Visual Validator Module을 위한 자동화 체크리스트 (QA 기준)

코다리가 스크립트 적용을 위해 필요한 구체적인 검증 항목을 아래와 같이 정의합니다. 이 항목들은 영상 에셋(썸네일, Intro 화면)이 생성될 때마다 자동으로 체크되어야 합니다.

| ID | 검증 항목 (Visual Check) | 요구 기준 (Rule) | 시스템 연관 요소 |
| :--- | :--- | :--- | :--- |
| **V1** | 배경 색상 준수 (Background Color) | **Deep Navy** 코드 (`#0A0A2A` 또는 최종 확정 코드) 사용 여부 | 컬러 팔레트 |
| **V2** | 강조 색상 적용 (Accent Color) | **Gold** 코드 (`#FFD700` 또는 최종 확정 코드)가 텍스트/라인에 최소 X% 이상 포함되었는지 확인 | 컬러 팔레트 |
| **V3** | 타이포그래피 크기 및 계층 (Font Sizing) | 헤드라인(H1)은 최소 Ypx, 본문(Body)은 최소 Zpx를 준수하는지 확인 | 타이포그래피 규칙 |
| **V4** | 인증 마크 배치 (Trust Mark Placement) | 인증 마크는 화면 우측 하단에서 X% 이상 떨어진 위치에 정확히 배치되었는지 확인 | 로고/마크 시스템 |
| **V5** | 대비 및 가독성 (Contrast Check) | 텍스트와 배경 간의 명도 대비(Contrast Ratio)가 WCAG AA 레벨 이상인지 확인 | 디자인 가이드라인 |

### 3. 자동 적용 방안 구체화 (Designer & Developer 협업 시나리오)

코다리가 `pack_apply.py` 등을 통해 이 QA 체크리스트를 영상 제작 파이프라인에 통합하기 위해 필요한 최종 통합 로드맵은 다음과 같습니다.

**단계 1: 시스템 정의 (Designer 역할 완료)**
*   최종 확정된 'Deep Navy/Gold' 색상 코드, 타이포그래피 규칙, Visual Sign-off 기준을 **단일 JSON 또는 YAML 파일**로 정리하여 개발팀에 제공합니다. (이것이 자동화의 입력값)

**단계 2: 모듈 설계 (Developer 역할)**
*   제공된 시스템 정의 파일을 읽어들이는 **Visual Validator Module**을 설계합니다.
*   이 모듈은 영상 에셋(썸네일 이미지, 오버레이 레이어 등)을 분석하여 V1부터 V5까지의 규칙을 검사하도록 로직을 구현합니다.

**단계 3: 파이프라인 통합 (Developer 역할)**
*   기존의 `pack_apply.py` 실행 흐름에 **Visual Validator Module** 호출 단계를 삽입하여, 에셋 생성 직후 자동 QA가 이루어지도록 파이프라인을 재구성합니다.
