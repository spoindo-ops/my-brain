## 1. 비주얼 시스템 정의 (Visual System Definition)

### A. 컬러 팔레트 (Color Palette)
| 명칭 | 역할 | HEX 코드 | RGB 값 | 적용 범위 |
| :--- | :--- | :--- | :--- | :--- |
| **Deep Navy** (배경/신뢰) | 메인 배경, 전문성, 신뢰 구축 | `#0A1931` | R: 10, G: 25, B: 49 | 전체 배경 및 주요 정보 블록 |
| **Gold Accent** (프리미엄/인증) | 강조 요소, 인증 마크, 핵심 CTA | `#FFD700` | R: 255, G: 215, B: 0 | 핵심 메시지, 하이라이트, 로고 및 인증 마크 |
| **Pure White** (텍스트/대비) | 본문 텍스트, 명확성 확보 | `#FFFFFF` | R: 255, G: 255, B: 255 | 보조 설명 및 대비가 필요한 텍스트 |

### B. 타이포그래피 정의 (Typography Definition)
| 용도 | 폰트명 (Font Name) | 스타일 | HEX 코드 | 적용 목적 |
| :--- | :--- | :--- | :--- | :--- |
| **헤드라인** | Montserrat Bold | 대문자, 두꺼운 굵기 | `#FFD700` (Gold) | 핵심 메시지 및 제목 강조 |
| **본문/설명** | Roboto Medium | 일반 텍스트 | `#FFFFFF` (White) | 보조 정보 및 QA 체크리스트 내용 |

### C. 레이아웃 가이드라인 (Layout Guideline - 썸네일 기준)
모든 썸네일은 다음의 원칙을 엄격히 준수해야 합니다:
1. **배경:** Deep Navy (`#0A1931`)를 기본으로 설정합니다.
2. **강조:** Gold 색상을 사용하여 모든 중요한 데이터 포인트와 인증 마크를 시각적으로 분리하고 부각시킵니다.
3. **대비:** 텍스트(White)는 Gold 강조 요소와 충분한 대비를 이루어야 하며, 정보의 계층 구조가 명확하게 구분되어야 합니다. (Navy 배경 위에서 Gold/White로 정보를 정렬)

## 2. 시각적 실행 가능성 QA 체크리스트 (Visual Sign-off Checklist)

Writer 또는 Editor는 아래 체크리스트를 모든 영상 에셋에 대해 검토하고 'Visual Sign-off' 기록을 남겨야 합니다. 이 프로세스는 **신뢰 자본** 구축의 데이터로 활용됩니다.

### ✅ 1단계: 시스템 일관성 점검 (System Consistency Check)
| 항목 | 기준 (Pass/Fail) | 확인 사항 | QA 상태 | 비고 (수정 요청) |
| :--- | :--- | :--- | :--- | :--- |
| **배경 적용** | Pass/Fail | 배경이 Deep Navy (`#0A1931`)인가? | | |
| **강조 색상** | Pass/Fail | Gold(`\#FFD700`)가 핵심 요소(인증, CTA)에만 사용되었는가? | | |
| **폰트 일관성** | Pass/Fail | Montserrat와 Roboto만 사용되었으며 톤앤매너가 유지되는가? | | |
| **레이아웃 정렬** | Pass/Fail | 모든 요소가 Deep Navy 배경 위에서 Gold/White로 명확하게 대비되며 정렬되어 있는가? | | |

### ✅ 2단계: 품질 증명 메시지 점검 (Quality Proof Message Check)
| 항목 | 기준 (Pass/Fail) | 확인 사항 | QA 상태 | 비고 (수정 요청) |
| :--- | :--- | :--- | :--- | :--- |
| **'품질 인증' 명확성** | Pass/Fail | 영상에서 '현지화된 품질 인증' 또는 관련 키워드가 시각적으로 명확하게 증명되는가? | | |
| **QA 요소 노출** | Pass/Fail | QA 체크리스트의 일부(예: 마크, 기준)가 자연스럽게 통합되어 있는가? | | |
| **신뢰도 연관성** | Pass/Fail | 시각적 디자인이 '깊고 전문적인 신뢰'를 전달하는가? (단순히 예쁜 것을 넘어선 증명력) | | |

### ✅ 3단계: 최종 승인 기록 (Final Sign-off Record)

| **코멘트 기록** | (구체적인 피드백 기록) | | |
