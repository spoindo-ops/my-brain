# 💎 Han Soo 비주얼 시스템 및 품질 인증 가이드라인 (Visual System & QA Guideline)

본 문서는 **Deep Navy/Gold** 시각 시스템을 기반으로 하며, 모든 콘텐츠에 적용될 색상 코드, 타이포그래피 규칙, 그리고 '현지화된 품질 인증'을 위한 Visual Sign-off 기준을 공식적으로 정의합니다. 이는 시장 신뢰 자본(Trust Capital) 구축 전략의 핵심 도구입니다.

---

## 1. 브랜드 시각 시스템 정의 (Visual System Definition)

**핵심 전략:** 시각적 일관성(Deep Navy/Gold)과 품질 인증(QA)을 통해 시장 신뢰를 확보한다.

### 1.1. 색상 팔레트 (Color Palette)

| 용도 | 색상 이름 | HEX 코드 | RGB 값 | 사용 목적 |
| :--- | :--- | :--- | :--- | :--- |
| **주요 배경** | Deep Navy | `#0A1931` | `(10, 25, 49)` | 콘텐츠의 주요 배경 및 안정감 부여 (신뢰 기반) |
| **강조 색상** | Gold Accent | `#FFD700` | `(255, 215, 0)` | 핵심 메시지 강조, 인증 마크, CTA 버튼 등 주목도 극대화 |
| **보조 텍스트** | Off-White | `#F5F5F5` | `(245, 245, 245)` | 본문 및 보조 정보 표시 (가독성 확보) |

### 1.2. 타이포그래피 규칙 (Typography Rules)

모든 텍스트는 Deep Navy 배경 위에서 Gold로 강조되어야 하며, 가독성을 최우선으로 한다.

| 요소 | 서체 (Font Family) | 스타일 적용 기준 | 용도 |
| :--- | :--- | :--- | :--- |
| **헤드라인 (H1/Title)** | Bold, Sans-serif 계열 (예: Montserrat Bold 또는 Poppins Bold) | Gold 색상으로 전체 텍스트를 감싸거나, 핵심 단어에만 적용. 최대 크기 사용. | 콘텐츠의 주제 및 후크 강조 |
| **본문 (Body Text)** | Regular, Sans-serif 계열 (예: Roboto Regular 또는 Noto Sans Regular) | Off-White 색상으로 명확하게 표시. 충분한 행간(Line Height) 확보. | 상세 정보 전달, 신뢰 증명 내용 |
| **인증/강조 마크** | Gold Accent | 특정 요소에만 적용하여 '품질 인증'의 시각적 무게를 부여. | Visual Sign-off 및 품질 강조 |

### 1.3. 디자인 가이드라인 (Application Guidelines)

1.  **배경 우선:** 모든 콘텐츠는 Deep Navy (`#0A1931`)을 기본 배경으로 설정한다.
2.  **강조 사용:** Gold (`#FFD700`)는 정보의 중요도에 따라 전략적으로 사용하며, 과도한 사용은 금지한다. (최대 30% 이내)
3.  **여백:** 충분한 여백을 확보하여 시각적 피로도를 줄이고 고급스러운 느낌을 유지한다.

---

## 2. 품질 인증 프로세스 (Visual Sign-off & QA Protocol)

모든 에셋은 다음의 통합된 QA 체크리스트를 통과해야 하며, Visual Sign-off 기록이 **'신뢰 자본'**으로 공식 인정된다.

### 2.1. Visual Sign-off 최종 체크리스트 (QA Criteria)

| 항목 | 검증 내용 | 기준 (Pass/Fail) | 담당자 |
| :--- | :--- | :--- | :--- |
| **A. 시각적 일관성** | Deep Navy / Gold 색상 코드가 100% 준수되었는가? | Pass / Fail | Designer |
| **B. 타이포그래피** | 정의된 폰트 규칙(H1, Body)이 적용되었으며 가독성이 확보되었는가? | Pass / Fail | Designer |
| **C. 메시지 일치** | 콘텐츠의 핵심 메시지가 시각적으로 명확하게 전달되는가? ('품질 증명' 강조 여부) | Pass / Fail | Writer/Designer |
| **D. 품질 인증 반영** | 필수 Visual Sign-off 마크 또는 인증 요소가 정확히 배치되었는가? | Pass / Fail | Designer |
| **E. 기술적 완성도** | 해상도, 텍스트 정렬, 에셋의 선명도가 요구 기준을 충족하는가? | Pass / Fail | Editor/Designer |

### 2.2. QA 실행 절차 (Execution Flow)

1.  **제작 단계 (Designer):** 모든 에셋 제작 후, 항목 A, B, D에 대해 **자체 검토 및 기록**을 수행한다.
2.  **검증 단계 (Editor):** 에셋 적용 후, 항목 C와 E를 중심으로 기술적/메시지적 정확성을 검증하고 기록한다.
3.  **최종 승인 (Sign-off):** 모든 체크리스트가 'Pass'일 경우에만 최종 **Visual Sign-off 기록**이 생성된다. 이 기록은 신뢰 자본 데이터로 활용된다.
