### **🛠️ 1. 사용된 주요 기술 스택**

이 애플리케이션은 **프론트엔드 컴퓨터 비전 기능과 AI 생성형 오디오 기술**을 결합한 매우 발전된 형태의 SPA(Single Page Application)입니다. 다음과 같은 기술들이 조화롭게 작동하고 있습니다.

1. **웹 카메라 및 비디오 처리 (HTML5 & Canvas API)**
    - navigator.mediaDevices.getUserMedia를 통해 실시간으로 사용자의 웹캠 영상을 가져옵니다.
    - <canvas> 요소를 통해 오버레이 (얼굴 랜드마크, 사물 인식 박스, 네온 효과 등)를 직접 렌더링합니다.
2. **클라이언트 사이드 머신러닝 (TensorFlow.js & MediaPipe)**
    - **객체 인식 (@tensorflow-models/coco-ssd):** 영상을 분석하여 노트북, 컵, 사람 등 주변 사물을 1프레임 단위로 실시간 탐지합니다. (영상 데이터를 서버로 보내지 않고 **브라우저 안에서 직접 연산**하여 빠르고 프라이버시가 보호됩니다).
    - **안면 인식 및 감정 분석 (@mediapipe/tasks-vision):** Face Landmarker를 이용해 478개의 안면 특징점(랜드마크)을 추적합니다. 미소, 찡그림, 눈 깜빡임 등 미세한 근육 움직임(Blendshapes) 기반 수치를 바탕으로 "행복", "공포", "놀람" 같은 감정 상태로 변환합니다.
3. **LLM (대형 언어 모델) 연동 (Google Gen AI SDK)**
    - 감지된 사물과 감정 데이터를 바탕으로 Gemini API(현재 gemini-flash-lite-latest 모델 사용)에 전달하여 실시간으로 짧고 추상적인 **사운드스케이프 프롬프트(예: "ethereal ambient drone, calm")**를 텍스트로 생성합니다.
4. **Google Lyria / RealTime Audio API (실험적 기능 연동)**
    - ai.live.music.connect를 호출하여 생성된 텍스트 프롬프트를 바탕으로 AI가 실시간으로 **원음 규격의 오디오 스트림(PCM)**을 끊임없이 생성하여 내려주는 기술을 사용합니다.
    - 내려받은 Base64 오디오 청크를 AudioContext를 통해 실시간으로 이어붙여 끊김 없이 스피커로 재생합니다(PCMPlayer 클래스).
5. **보조 오디오 엔진 (Tone.js)**
    - 만약 실시간 오디오 생성 API의 연결이 불안정할 경우를 대비하여(또는 부드러운 전환을 위해) 웹 브라우저 자체의 오실레이터(발진기)를 활용한 자체 **절차적 신디사이저 엔진(ProceduralMusicEngine)**이 내장되어 있습니다. 감정에 따라 음계, 박자, 파형을 스스로 생성합니다.
6. **UI/UX (React & Tailwind CSS, Motion)**
    - React State 리렌더링을 최적화하기 위해, 초당 수십 번 발생하는 비전 모델의 결과를 useRef로 관리하며 10fps 정도로만 UI 상태에 갱신하여 렌더링 부하를 줄였습니다.
    - Tailwind CSS로 전체적인 디자인 시스템과 Glassmorphism(유리 질감) 효과를 구성했습니다.

### **💬 2. 이 웹앱을 생성하기 위해 입력해야 하는 프롬프트**

이 앱을 처음부터 다시 만들어달라고 AI에게 지시한다면, 단순히 "음악 앱 만들어줘"가 아니라 아주 구체적인 시나리오와 모델 통합 지시가 들어가야 합니다.

만약 저(AI Studio 에이전트)에게 다음 프로젝트를 시작할 때 아래와 같이 지시하시면 이런 결과를 얻을 수 있습니다:

> **[지시 프롬프트 예시]**
> 
> - "CONNECT AI LAB"이라는 이름의 AI 프로젝트를 만들어줘. 이 앱은 **카메라를 통한 컴퓨터 비전 분석 결과를 바탕으로, AI가 주변 환경과 사용자의 기분에 맞는 앰비언트 음악을 실시간으로 끊임없이 연주해 주는 애플리케이션**이야.
> 
> **디자인 요구사항:**
> 
> - 가로/세로 화면에 모두 대응하는 대시보드 형태의 풀스크린으로 만들어줘.
> - 색상은 딥 블루/스카이 계열을 사용하고, 모던한 Glassmorphism(유리 질감) 카드 형태로 컴포넌트들을 구성해 줘. 언어는 모두 한국어로 써줘. UI에 스캐너가 돌아가는 듯한 레이더 애니메이션을 백그라운드로 띄워줘.
> 
> **기능 요구사항:**
> 
> 1. 브라우저에서 사용자의 카메라 접근 권한을 받고, 영상을 배경에 어둡게 깔아줘.
> 2. **객체 인식:** @tensorflow-models/coco-ssd를 사용해서 카메라에서 감지되는 사물(Cup, Laptop, Person 등)을 실시간으로 박스를 쳐서 화면에 시각적으로 그려주고, 좌측 패널에 리스트업해 줘.
> 3. **안면 인식:** @mediapipe/tasks-vision의 FaceLandmarker를 이용해 얼굴을 스캔하고, 랜드마크 점들을 캔버스에 사이버펑크 느낌으로 덧그려줘. 그리고 Face Blendshapes 데이터를 측정해서 사용자의 현재 감정(행복, 분노, 슬픔, 평온 등)을 도출하고 화면에 퍼센티지로 표시해 줘.
> 4. **프롬프트 엔진:** 3~5초마다 현재 감정 상태와 감지된 사물 리스트를 GoogleGenAI 모듈을 통해 Gemini를 호출하여 짧은 사운드스케이프 프롬프트(예: "조용한 숲속의 앰비언트")를 텍스트로 생성해 줘.
> 5. **오디오 엔진:** 생성된 그 프롬프트를 바탕으로 ai.live.music.connect API (Lyria RealTime Exp 모델)를 호출해 생성되는 Base64 오디오 청크를 PCM 형태로 실시간 조립해서 끊김 없이 재생해줘. 오디오 재생 플레이어 클래스도 직접 구현해 줘.

이처럼 **1) 목적, 2) 디자인 테마 (색상 및 언어), 3) 사용할 외부 라이브러리와 API의 역할**을 조목조목 나누어 지시하면 AI가 훨씬 더 정교하고 완전한 결과물을 단번에 만들어낼 수 있습니다.
